# Security Policy

Most issues in this repository are documentation defects, but some documentation mistakes can still create real risk if they cause unsafe deployments, key handling, or billing behavior.

## Report a vulnerability

Do not open public issues or pull requests for security-sensitive findings.

Preferred reporting path:

1. Open a private GitHub security advisory: [vowena/docs security advisories](https://github.com/vowena/docs/security/advisories/new)
2. Or email `security@vowena.xyz` with the subject `[SECURITY] docs vulnerability report`

## Examples of in-scope issues

- Instructions that could cause loss of funds or unsafe contract deployment
- Leaked credentials, secrets, or private endpoints in examples
- Malicious or unsafe copy-paste snippets
- Broken security guidance that could put users at risk

## Response targets

- Acknowledgment within 48 hours
- Initial assessment within 7 days

## Coordinated disclosure

Please allow time for verification and remediation before public disclosure.
